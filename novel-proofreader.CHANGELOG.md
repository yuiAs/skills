# Changelogs

## 2026-05-31

`★ Insight ─────────────────────────────────────`
**LLM にスキルを確実に実行させるための設計原則:**
- 「参照して適用」のような抽象指示は「目を通したけどやらない」になりがち
- 「Grep で検出 → 修正 → Read で検証」のように **具体的なツール操作を指定** すると実行率が上がる
- 「創作物の尊重」のような免責条項は、版組（機械的修正）と校正（内容判断）で **分離** しないと、機械的修正まで免除される
`─────────────────────────────────────────────────`

### 変更点サマリ

| 箇所                     | Before                             | After                                                                                    |
| ------------------------ | ---------------------------------- | ---------------------------------------------------------------------------------------- |
| **事前検出**             | なし                               | **ステップ3 として必須化**。Grep で記号・空行・読点密度を検出してから修正に入る          |
| **版組修正の強制力**     | 「機械的に修正します」(宣言のみ)   | **「スキップ禁止」を明記**。検出→修正→検証の3段階を義務化                                |
| **処理の流れ**           | 5ステップ (抽象的)                 | **7ステップ (具体的)**。検出・検証を独立ステップとして追加                               |
| **出力検証**             | なし                               | **ステップ5 として必須化**。出力ファイルの冒頭を Read して修正反映を確認                 |
| **注意事項**             | 版組も校正も一括で「創作物の尊重」 | **版組と校正を分離**。版組は「作者の意図を理由にスキップしない」、校正は「創作物の尊重」 |
| **複数ファイル処理**     | 言及なし                           | Agent ツールでの並列処理の方法を追加                                                     |
| **各カテゴリの検出方法** | references に丸投げ                | SKILL 本体に **具体的な Grep パターン** を記載                                           |
